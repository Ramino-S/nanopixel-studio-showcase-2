## 🛠️ Архитектурные решения и паттерны в коде

Для демонстрации были выбраны 5 основных файлов, отражающих подход к проектированию отказоустойчивых веб-приложений.

### 1. Каскадное резервирование (Паттерны Facade и Fallback Cascade)
**Файл:** [`services/aiService.ts`](./services/aiService.ts)  
Этот сервис выступает единым фасадом для работы с генеративными моделями, полностью скрывая сетевую логику от компонентов интерфейса.
*   **Отказоустойчивость:** Если основная выбранная модель возвращает ошибку (например, из-за превышения лимитов API или недоступности провайдера), сервис автоматически перебирает цепочку альтернативных моделей в фоновом режиме (например, с более мощной Flux Pro на легковесную Flux Klein), предотвращая сбой в пользовательском интерфейсе.

### 2. Параллельные запросы и авто-повторы (Concurrency & Retries)
**Файл:** [`services/openRouterService.ts`](./services/openRouterService.ts)  
Интеграция с API-интерфейсом OpenRouter для генерации и редактирования изображений.
*   **Parallelism:** При запросе нескольких вариантов изображений (`count > 1`) запросы выполняются параллельно с помощью `Promise.allSettled()`, что ускоряет получение результатов до 2-4 раз по сравнению с последовательным циклом.
*   **Resilience (Ретраи):** Реализован механизм автоматического повтора (1 попытка через 1500мс) при получении сбоев или пустых ответов.
*   **Мягкая деградация:** Если часть параллельных генераций завершилась успешно, сервис вернет готовые изображения, записав предупреждения о неудавшихся в отладочный журнал, без аварийного завершения работы.

### 3. Буферизованное логирование на клиенте (Observability)
**Файл:** [`services/logService.ts`](./services/logService.ts)  
Кастомное решение для сбора телеметрии и отладки работы приложения на стороне клиента.
*   **Безопасность localStorage:** Логи сохраняются в локальное хранилище. При достижении лимита по объёму памяти логгер перехватывает ошибку переполнения (`QuotaExceededError`), безопасно очищает половину старых записей и сохраняет новые данные.
*   **Реактивность:** Реализован паттерн «Подписка» (Observer), позволяющий визуальным компонентам моментально реагировать на новые логи без интервальных проверок.

### 4. Реактивный просмотрщик отладки (React UI)
**Файл:** [`components/LogViewer.tsx`](./components/LogViewer.tsx)  
Интерактивная панель журналов событий.
*   **Управление подписками:** Компонент подписывается на обновления логов только при монтировании и отписывается при закрытии, гарантируя отсутствие утечек памяти.
*   **Эргономика:** Реализован автоматический скролл к последней строке событий, интерактивные фильтры по уровням (`DEBUG`, `INFO`, `WARN`, `ERROR`) и функция скачивания логов в формате JSON/TXT.

### 5. Контракты данных (TypeScript Domain)
**Файл:** [`types.ts`](./types.ts)  
Все перечисления, конфигурации моделей, форматы истории и типы генерации строго типизированы, исключая применение нетипизированного типа `any`.
